Display of a static page in Django

The MVT model

Django follows the MVT model : Model-View-Template

  • Model : Handles data
  • View : Contains the logic
  • Template : Defines the HTML presentation

Flow from a request to a response

  • The URL dispatcher matches the URL and calls the corresponding view
  • The view handles the necessary logic to display the page like db calls (minimal when the page is static)
  • The view prepares the context data and select the template to use
  • Django template engine loads the template, loads it with context data and generates the HTML
  • Django then returns an HttpResponse containing the HTML to the client browser

How to call template rendering ?

The most common way is to call the render() method. It accepts the request, a path to a template file and the context to be displayed.

A class view can also be used (inheritance from ListView). It takes the same inputs with a object syntax.
You can also call the primitives methods used by the render() method.
If you return an HttpResponse from a loaded template, Django will render it. This can work for static pages but scalability will suffer.

Summary

  • View receives request
  • View prepares context
  • render() call
  • Template loader finds and loads the template
  • Template engine processes the template {% templatetag openvariable %} {% templatetag closevariable %} variables and {% templatetag openblock %} {% templatetag closeblock %} tags)
  • Final HTML string is generated
  • HttpResponse is sent to the client
